React + webpack 配置教程

最近我在学习React并且试着找出一个好的配置环境,找了很多教程,其中也不乏有些问题,最后决定用webpack作为我的构建工具,现在跟大家分享一下我学习到的经验.

初始化一个项目

由于React没有任何工具来生成和管理项目,我们就要自己处理所有的事情.需要创建目录,初始化Git仓库,设置npm…

运行完npm init之后,建议在package.json文件里面加上"private": true,这样就不会把我们的项目意外发布到npm上面了.

设置webpack

webpack就是一个构建工具,将我们的应用代码生成静态资源文件.相比其他构建工具webpack有很多优势,比如可以剔除无用的代码,热更新模块,配置起来简单方便易操作.
在开始之前我们需要同时全局和在项目内安装webpack,全局安装是为了webpack命令可用,项目内安装可以让我们指定我们需要的版本,而不去依赖于全局安装的版本.
